# 加价结算助手 - 界面设计方案

## 应用概述
一款用于管理加价结算明细的移动工具应用，帮助用户记录、跟踪和统计加价订单的结算情况。

## 屏幕列表

### 1. 首页（Dashboard）
- Tab 1: 展示今日/总体统计概览
- 顶部显示关键数据卡片：总订单数、已转金额、未转金额、总实际到手
- 下方显示最近订单列表（最新5条）
- 快速新增按钮（浮动按钮）

### 2. 订单列表页（Orders）
- Tab 2: 完整订单列表
- 顶部筛选栏：按转账状态（全部/已转/未转）筛选
- 搜索功能：按群名搜索
- 每条订单卡片显示：群名、加价后总价、加价、转账状态标签
- 点击进入订单详情

### 3. 订单详情/编辑页（Order Detail）
- 模态页面，展示订单所有字段
- 支持编辑所有字段
- 支持删除订单
- 支持修改转账状态

### 4. 新增订单页（Add Order）
- 模态表单页面
- 输入字段：接单日期、单号、群名、原价、加价后总价
- 自动计算：加价 = 加价后总价 - 原价
- 自动计算：原价应到手 = 原价 × 40%
- 自动计算：加价应到手 = 加价 × 40%
- 手动输入：实际转出、转账状态

### 5. 统计页（Statistics）
- Tab 3: 数据统计汇总
- 总收入统计、已结算/未结算金额
- 按转账状态分组统计
- 平均每单加价、最高加价等

## 主要内容和功能

### 数据字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 序号 | number | 自动递增 |
| 接单日期 | date | 订单日期 |
| 单号 | string | 订单编号 |
| 群名 | string | 客户/群名称 |
| 原价 | number | 商品原价 |
| 原价应到手 | number | 原价×40% |
| 加价后总价 | number | 加价后总价 |
| 加价 | number | 加价金额 |
| 加价应到手 | number | 加价×40% |
| 应转出 | number | 应转出金额 |
| 实际转出 | number | 实际转出金额 |
| 转账状态 | enum | 已转/未转 |
| 加价实际到手 | number | 加价实际到手 |
| 实际到手 | number | 总实际到手 |
| 登记状态 | string | 登记状态 |
| 结算状态 | string | 结算状态 |

## 关键用户流程

### 新增订单流程
1. 用户点击首页浮动"+"按钮
2. 弹出新增订单表单
3. 输入群名、原价、加价后总价等
4. 系统自动计算加价、应到手等字段
5. 用户确认保存
6. 返回列表，新订单出现在顶部

### 查看/编辑订单流程
1. 用户在订单列表点击某条订单
2. 进入订单详情页
3. 可查看所有字段详情
4. 点击编辑进入编辑模式
5. 修改后保存

### 筛选订单流程
1. 在订单列表页顶部选择筛选条件
2. 选择转账状态：全部/已转/未转
3. 或输入群名搜索
4. 列表实时更新

## 设计原则
- 遵循 Apple HIG 设计规范
- 竖屏模式，单手操作友好
- 大字号金额显示，易于阅读
- 状态标签用颜色区分（绿色=已转，红色=未转）
- 卡片式布局，信息层次清晰
